Робота зі статичними файлами

Статичні файли - ресурси сайту, що не змінюються при перезапитах - картинки, файли стилів та скриптів, звуки, відео тощо. Зазвичай, якщо запит містить ім'я існуючого файлу, то сервер автоматично завантажує його у відповідь. Проте, формально, запит не прямо звертається до файлу, а це робить сервер при відповідних налаштуваннях. У Django автоматичного вивантаження файлів немає, необхідно змінювати налаштування серверу.

  • Знаходимо та відкриваємо файл налаштувань проєкту (/Project33/settings.py)
  • Знаходимо розділ з коментарем # Static files (CSS, JavaScript, Images) Додаємо інструкції
    STATICFILES_DIRS = [
      BASE_DIR / "App33/static"
    ]
  • Створюмо директорію App33/static (папку static у директорії застосунку App3). У ній - піддиректорію css, у ній - файл site.css
  • Вносимо зміни до _layout.html - до початку додаємо { % load static % }, у заголовкову частину - додаємо посилання на стилі
    <link rel="stylesheet" href="{% static 'css/site.css' %}" />
  • Оновлюємо сторінку на браузері, спостерігаємо за підключенням стилей
  • Директиву { % load static % } повторюємо у всіх файлах-шаблонах, у яких є звернення до ресурсів.
